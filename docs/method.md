# Методика работы над проектом

## Общий подход
Работа над проектом информационной системы учёта склада выполняется поэтапно. Сначала формулируется проблема и цель, затем создаётся структура репозитория, описываются складские процессы, модель данных, прототип интерфейса и проверки.

## Основные этапы
1. Анализ предметной области склада интернет-магазина.
2. Описание пользователей и основных сценариев.
3. Проектирование структуры данных.
4. Подготовка прототипа интерфейса.
5. Описание тестирования и релизного процесса.
6. Фиксация изменений в GitHub.

## Почему используется GitHub
GitHub позволяет хранить историю изменений, работать с ветками, проверять изменения через Pull Request и фиксировать контрольные версии проекта.
