# Информационная система учёта склада

## Назначение проекта
Проект посвящён созданию информационной системы учёта склада для интернет-магазина или маркетплейса. Система помогает контролировать поступление товаров, остатки, заказы, резервирование, отгрузку, возвраты и инвентаризацию.

Главная задача проекта — снизить количество ошибок ручного учёта и сделать складские операции более прозрачными.

## Состав репозитория
- docs/ — проектная документация, постановка задачи и методика;
- src/ — исходные файлы, прототипы или скрипты;
- data/sample/ — небольшой пример демонстрационных данных;
- data/raw/ — локальные исходные данные, которые не отправляются на GitHub;
- experiments/ — журнал проверок или экспериментов;
- assets/ — изображения, схемы и диаграммы;
- sql/ — SQL-скрипты для базы данных;
- migrations/ — миграции структуры базы данных;
- tests/ — тестовые сценарии;
- design/ — материалы прототипа интерфейса.

## План выполнения
1. Постановка задачи и анализ предметной области.
2. Описание требований и пользователей системы.
3. Разработка модели данных и схем складских процессов.
4. Подготовка прототипа интерфейса.
5. Проверка результата и тестирование.
6. Подготовка пояснительной записки и выводов.
